# Configuration Reference

Everything that goes inside the `starlight({ … })` call in `astro.config.mjs`, plus the content-collection helpers from `@astrojs/starlight/loaders` and `@astrojs/starlight/schema`.

```js
// astro.config.mjs
import { defineConfig } from 'astro/config';
import starlight from '@astrojs/starlight';

export default defineConfig({
  integrations: [
    starlight({
      title: 'My delightful docs site',
      // …options below
    }),
  ],
});
```

## All `starlight()` options

### `title` (required)

**type:** `string | Record<string, string>`

Site title (used in `<title>` and metadata). For multilingual sites, pass an object keyed by BCP-47 tag:

```js
title: { en: 'My docs', de: 'Meine Dokumentation' }
```

### `description`

**type:** `string`

Default site description used in `<meta name="description">` when no per-page description is set.

### `logo`

**type:** `LogoConfig`

```ts
type LogoConfig = { alt?: string; replacesTitle?: boolean } &
  ({ src: string } | { light: string; dark: string });
```

A single `src` is used for both color modes; `light` / `dark` lets you swap variants. `replacesTitle: true` visually hides the site title text while keeping it for screen readers.

```js
logo: {
  src: './src/assets/my-logo.svg',
  replacesTitle: true,
}
```

### `tableOfContents`

**type:** `false | { minHeadingLevel?: number; maxHeadingLevel?: number }`
**default:** `{ minHeadingLevel: 2, maxHeadingLevel: 3 }`

Site-wide TOC settings. Set `false` to disable the right-hand TOC entirely. Overridable per page via frontmatter.

### `editLink`

**type:** `{ baseUrl: string }`

Enable an "Edit page" link in the footer. `baseUrl` is prepended to the path of each page:

```js
editLink: {
  baseUrl: 'https://github.com/withastro/starlight/edit/main/',
}
```

Common patterns:
- GitHub: `https://github.com/USER/REPO/edit/BRANCH/`
- GitLab: `https://gitlab.com/USER/REPO/-/edit/BRANCH/`

If the docs aren't at the repo root, include the path: `https://github.com/USER/REPO/edit/main/docs/`.

### `sidebar`

**type:** `SidebarItem[]`

Top-level sidebar configuration. See `references/navigation-and-search.md` for the full sidebar grammar (groups, autogenerate, badges, slug shorthand, translations).

Quick shape:

```js
sidebar: [
  // String shorthand = internal slug
  'getting-started',

  // Internal link with override
  { slug: 'guides/intro', label: 'Introduction' },

  // External / non-docs link
  { label: 'GitHub', link: 'https://github.com/example/repo' },

  // Manual group
  {
    label: 'Start Here',
    items: ['intro', 'installation'],
  },

  // Autogenerated group
  {
    label: 'Reference',
    autogenerate: { directory: 'reference', collapsed: true },
  },
]
```

`SidebarItem` formal types:

```ts
type SidebarItem =
  | string
  | ({ translations?: Record<string, string>; badge?: string | BadgeConfig } & (
      | { link: string; label: string; attrs?: Record<string, …> }            // External link
      | { slug: string; label?: string; attrs?: Record<string, …> }            // Internal link
      | { label: string; items: SidebarItem[]; collapsed?: boolean }           // Group
      | {                                                                       // Autogenerated group
          label: string;
          autogenerate: { directory: string; collapsed?: boolean; attrs?: Record<string, …> };
          collapsed?: boolean;
        }
    ));

interface BadgeConfig {
  text: string;
  variant?: 'note' | 'tip' | 'caution' | 'danger' | 'success' | 'default';
  class?: string;
}
```

### `locales`

**type:** `{ [dir: string]: LocaleConfig }`

Enable multilingual mode. Each key is a directory name under `src/content/docs/`.

```ts
interface LocaleConfig {
  label: string;       // Display label shown in the language switcher.
  lang?: string;       // BCP-47 tag (defaults to the directory name).
  dir?: 'ltr' | 'rtl'; // Writing direction (default 'ltr').
}
```

```js
locales: {
  en: { label: 'English' },
  'zh-cn': { label: '简体中文', lang: 'zh-CN' },
  ar: { label: 'العربية', dir: 'rtl' },
}
```

To serve the default language without a `/lang/` prefix, use the special key `root`:

```js
locales: {
  root: { label: 'English', lang: 'en' },  // 'lang' required for root
  fr: { label: 'Français' },
}
```

For a monolingual site in a non-English language, define only a `root` locale with the desired `lang`. See `references/i18n.md` for the full i18n workflow.

### `defaultLocale`

**type:** `string`

Which locale to fall back to for missing translations and UI labels. Must match a key in `locales`. (Skip this if your default is the `root` locale.)

### `social`

**type:** `Array<{ label: string; icon: StarlightIcon; href: string }>`

Social media links in the site header. Each entry needs an `icon` from the built-in icon set, a `label` (for screen readers), and an `href`.

```js
social: [
  { icon: 'discord', label: 'Discord', href: 'https://astro.build/chat' },
  { icon: 'github', label: 'GitHub', href: 'https://github.com/withastro/starlight' },
  { icon: 'mastodon', label: 'Mastodon', href: 'https://m.webtoo.ls/@astro' },
]
```

### `customCss`

**type:** `string[]`

Additional CSS files. Local paths (`'./src/styles/custom.css'`) and npm modules (`'@fontsource/roboto'`) both work. See `references/styling.md` for theming details and the CSS custom properties Starlight exposes.

### `markdown`

**type:** `{ headingLinks?: boolean; processedDirs?: string[] }`
**default:** `{ headingLinks: true, processedDirs: [] }`

- `headingLinks` — when `true` (default), headings get clickable anchor links. Set `false` to disable.
- `processedDirs` — extra directories to run through Starlight's Markdown pipeline (asides syntax, heading anchors, RTL code-block support). Useful when rendering content from a non-`docs` collection inside `<StarlightPage>`.

### `expressiveCode`

**type:** `StarlightExpressiveCodeOptions | boolean`
**default:** `true`

Controls Expressive Code (Starlight's syntax-highlighting and code-block UI library). Set to `false` to disable Expressive Code entirely.

Accepts all standard [Expressive Code options](https://expressive-code.com/reference/configuration/) plus three Starlight-specific:

- `themes` (default `['starlight-dark', 'starlight-light']`) — theme(s) used to highlight code. Provide at least one dark + one light to enable automatic theme switching.
- `useStarlightDarkModeSwitch` (default `true`) — when `true`, code-block theme follows the site theme switch.
- `useStarlightUiThemeColors` (default `true` if `themes` is unset, else `false`) — when `true`, code-block UI colors use Starlight's CSS variables.

Common tweak — rounded corners:

```js
expressiveCode: {
  styleOverrides: { borderRadius: '0.5rem' },
}
```

### `pagefind`

**type:** `boolean | PagefindOptions`
**default:** `true`

Default search. `false` disables Pagefind and hides the search UI. Object form configures the client; see `references/navigation-and-search.md` for the full `PagefindOptions` shape, ranking customization, and multi-site indexes.

Pagefind cannot be enabled when `prerender: false` (it requires built static HTML to index).

### `prerender`

**type:** `boolean`
**default:** `true`

When `false`, Starlight pages are rendered on-demand by an SSR adapter instead of being pre-rendered. Requires the Astro project to be configured for SSR.

### `head`

**type:** `HeadConfig[]`

Inject extra `<head>` tags site-wide. Common use cases: analytics scripts, additional meta tags, alternate icons.

```ts
interface HeadConfig {
  tag: string;
  attrs?: Record<string, string | boolean | undefined>;
  content?: string;
}
```

```js
head: [
  {
    tag: 'script',
    attrs: {
      src: 'https://cdn.usefathom.com/script.js',
      'data-site': 'MY-FATHOM-ID',
      defer: true,
    },
  },
]
```

Note: `head` entries become HTML directly — they bypass Astro's script/style processing. If you need to import local assets (bundled JS, processed CSS), [override the `Head` component](customization.md) instead.

### `lastUpdated`

**type:** `boolean`
**default:** `false`

When `true`, footer shows when each page was last updated. By default, the timestamp is read from Git history, so shallow clones (some deployment platforms do this) may give wrong dates. A page can override the date via the `lastUpdated` frontmatter field.

### `pagination`

**type:** `boolean`
**default:** `true`

Show prev/next page links in the footer. Override per page with `prev` / `next` frontmatter.

### `favicon`

**type:** `string`
**default:** `'/favicon.svg'`

Path (under `public/`) to the favicon. Must be a valid icon file (`.ico`, `.gif`, `.jpg`, `.png`, or `.svg`).

For multiple favicon formats (e.g. an ICO fallback for Safari), use `favicon` for the primary and add additional `<link rel="icon">` tags via `head:`.

### `titleDelimiter`

**type:** `string`
**default:** `'|'`

The separator in `<title>` tags between page title and site title. Default produces `Page Title | Site Title`.

### `disable404Route`

**type:** `boolean`
**default:** `false`

Set `true` to disable Starlight's default 404 page so you can use a fully custom `src/pages/404.astro` route. For minor 404 customization, you can instead just add `src/content/docs/404.md` (or `.mdx`) and use Starlight's frontmatter — typically with `template: splash` and a `hero:`.

### `routeMiddleware`

**type:** `string | string[]`

Path(s) to route middleware that can modify Starlight's per-page route data. Distinct from Astro's middleware. See `references/customization.md`.

### `components`

**type:** `Record<string, string>`

Paths to component files that replace Starlight's built-in components. Keys are the official component names (e.g. `SocialIcons`, `Footer`, `PageTitle`). See `references/customization.md` for the workflow and the full list of overridable components.

```js
components: {
  SocialIcons: './src/components/MySocialLinks.astro',
}
```

### `plugins`

**type:** `StarlightPlugin[]`

Custom plugins that extend Starlight at the integration level. See `references/plugins.md` for the plugin authoring API and `https://starlight.astro.build/resources/plugins/` for the community catalog.

### `credits`

**type:** `boolean`
**default:** `false`

When `true`, adds a "Built with Starlight" link to the footer.

## Content collection helpers

Starlight uses Astro content collections to load Markdown / MDX / Markdoc files and translation data. Configure them in `src/content.config.ts`:

```ts
import { defineCollection } from 'astro:content';
import { docsLoader, i18nLoader } from '@astrojs/starlight/loaders';
import { docsSchema, i18nSchema } from '@astrojs/starlight/schema';

export const collections = {
  docs: defineCollection({ loader: docsLoader(), schema: docsSchema() }),
  // Optional: enables UI translation strings for multilingual sites.
  i18n: defineCollection({ loader: i18nLoader(), schema: i18nSchema() }),
};
```

### Loaders (from `@astrojs/starlight/loaders`)

#### `docsLoader()`

Loads `.md` / `.mdx` / `.mdoc` files from `src/content/docs/`. Files starting with `_` are ignored.

Options:

- `generateId({ entry, base, data })` — replaces the default slug sluggifier. By default, `Example.File.md` becomes `/examplefile`. To preserve special characters, supply your own. For example, to keep the original casing and dots:

  ```ts
  docsLoader({
    generateId: ({ entry }) => entry.split('.').slice(0, -1).join('.'),
  }),
  ```

  See [`generateId()` in the Astro docs](https://docs.astro.build/en/reference/content-loader-reference/#generateid) for the full contract.

#### `i18nLoader()`

Loads JSON / YAML files from `src/content/i18n/`. Files starting with `_` are ignored. No options currently.

### Schemas (from `@astrojs/starlight/schema`)

#### `docsSchema()`

Parses frontmatter for the `docs` collection. Options:

- `extend` — a Zod schema (or function returning one) to merge into the built-in schema. Used to add custom frontmatter fields or tighten built-in ones. See `references/content-authoring.md` for examples.

#### `i18nSchema()`

Parses translation files. Options:

- `extend` — a Zod object to merge into the i18n schema. Used to declare custom translation keys for your site or for plugins.
