# Navigation and Search

This covers the sidebar (the main navigation) and the search experience (Pagefind by default, with DocSearch as an alternative).

## Sidebar basics

Configure the sidebar via the `sidebar` option in `starlight()`. Without configuration, Starlight autogenerates a sidebar from the directory structure of `src/content/docs/`, using each page's `title` frontmatter as the label.

A sidebar is an array of items. Each item is one of these shapes:

### Internal link

Points to a page in `src/content/docs/`. The page's `title` is the default label.

```js
sidebar: [
  { slug: 'constellations/andromeda' },
  { slug: 'constellations/orion', label: 'Orion (override)' },
]
```

Shorthand: just pass the slug as a string.

```js
sidebar: ['constellations/andromeda', 'constellations/orion']
```

In multilingual sites, `slug` does *not* include the locale prefix — `intro`, not `en/intro` or `pt-br/intro`. Starlight resolves the right localized page automatically.

### External or non-docs link

Use `link` + `label` for anything outside the docs collection.

```js
sidebar: [
  { label: 'Meteor Store', link: '/shop/' },       // non-docs page on this site
  { label: 'NASA', link: 'https://www.nasa.gov/' }, // external URL
]
```

### Manual group

Group related links under a collapsible heading.

```js
sidebar: [
  {
    label: 'Constellations',
    items: [
      'constellations/carina',
      'constellations/centaurus',
      {
        label: 'Seasonal',
        items: [
          'constellations/andromeda',
          'constellations/orion',
          'constellations/ursa-minor',
        ],
      },
    ],
  },
]
```

Groups can nest arbitrarily.

### Autogenerated group

Generate a group from a directory of docs files automatically — useful when you don't want to enumerate every page by hand.

```js
sidebar: [
  {
    label: 'Constellations',
    autogenerate: { directory: 'constellations' },
  },
]
```

The `directory` path is relative to `src/content/docs/`. Pages are sorted by file `id` alphabetically; override with the per-page `sidebar.order` frontmatter field.

Subdirectories inside the autogenerated directory become nested subgroups whose labels match the directory name (e.g. a `seasonal/` subdirectory shows up as a `seasonal` group).

## Customizing autogenerated links via frontmatter

Pages in autogenerated groups can shape how they appear via their `sidebar` frontmatter block:

```yaml
---
title: My page
sidebar:
  label: Custom sidebar label   # overrides title
  order: 2                      # lower → higher in the group
  hidden: false                 # exclude from sidebar entirely if true
  badge:                        # add a badge
    text: New
    variant: tip
---
```

These frontmatter overrides only affect autogenerated groups and docs items defined via `slug`. They don't apply to items configured via `link`.

## Collapsing groups

`collapsed: true` makes a group collapsed by default.

```js
sidebar: [
  {
    label: 'Constellations',
    collapsed: true,
    items: ['constellations/andromeda', 'constellations/orion'],
  },
]
```

For autogenerated groups, subgroups inherit the parent's `collapsed` value unless `autogenerate.collapsed` is set explicitly:

```js
sidebar: [
  {
    label: 'Constellations',
    collapsed: false,                          // top group open
    autogenerate: {
      directory: 'constellations',
      collapsed: true,                          // but its subgroups closed
    },
  },
]
```

## Badges on sidebar entries

Most sidebar shapes accept a `badge` property — a string for the default accent variant, or an object for full control:

```js
sidebar: [
  {
    label: 'Stars',
    items: [
      { slug: 'stars/persei', badge: 'Supergiant' },                          // string
      { slug: 'stars/sirius', badge: { text: 'Stub', variant: 'caution' } }, // object
    ],
  },
  {
    label: 'Moons',
    badge: 'Outdated',                                                        // on the group itself
    autogenerate: { directory: 'moons' },
  },
]
```

Badge `variant` options: `'note'` (blue), `'tip'` (purple), `'caution'` (yellow), `'danger'` (red), `'success'` (green), `'default'` (theme accent). A `class` property lets you apply additional CSS class names.

## Custom HTML attributes on sidebar links

The `attrs` property adds arbitrary HTML attributes to the rendered `<a>` element. Useful for opening external links in new tabs or applying inline styles.

```js
sidebar: [
  {
    label: 'Resources',
    items: [
      {
        label: 'NASA',
        link: 'https://www.nasa.gov/',
        attrs: { target: '_blank', style: 'font-style: italic' },
      },
    ],
  },
]
```

For autogenerated groups, you can apply attrs to all links in the group at once:

```js
sidebar: [
  {
    label: 'Constellations',
    autogenerate: {
      directory: 'constellations',
      attrs: { style: 'font-style: italic' },
    },
  },
]
```

Per-page `sidebar.attrs` frontmatter merges with the group's `autogenerate.attrs`.

## Translating sidebar labels

`translations` on a link or group provides labels for additional locales (keyed by BCP-47 tag). The base `label` is used for the default locale and as a fallback.

```js
sidebar: [
  {
    label: 'Constellations',
    translations: { 'pt-BR': 'Constelações' },
    items: [
      {
        slug: 'constellations/andromeda',
        translations: { 'pt-BR': 'Andrômeda' },
      },
    ],
  },
]
```

If you use `slug`-based internal links, translated page titles from the per-locale frontmatter are picked up automatically — no need to repeat them in `translations`.

Badge text can also be localized — pass an object instead of a string:

```js
{
  slug: 'constellations/andromeda',
  badge: {
    text: { en: 'New', 'pt-BR': 'Novo' },
  },
}
```

## Search (Pagefind, default)

Starlight ships with [Pagefind](https://pagefind.app/) for full-text search. No configuration required — build and deploy, and the search bar in the header just works.

Important constraint: Pagefind requires built static HTML to index. It cannot run when `prerender: false` (i.e. when Starlight pages are SSR'd).

### Hiding content from search

Exclude a whole page via frontmatter:

```yaml
---
title: Content to hide from search
pagefind: false
---
```

Exclude part of a page by wrapping it in an element with `data-pagefind-ignore`:

```mdx
---
title: Partially indexed page
---

This text will be discoverable via search.

<div data-pagefind-ignore>
This text will be hidden from search.
</div>
```

### Configuring Pagefind

The `pagefind` integration option accepts `true` (default), `false`, or a `PagefindOptions` object:

```ts
interface PagefindOptions {
  ranking?: {
    pageLength?: number;
    termFrequency?: number;
    termSaturation?: number;
    termSimilarity?: number;
  };
  indexWeight?: number;
  mergeIndex?: Array<{
    bundlePath: string;
    indexWeight?: number;
    basePath?: string;
    baseUrl?: string;
    mergeFilter?: Record<string, string | string[]>;
    language?: string;
    ranking?: { /* same fields as above */ };
  }>;
}
```

- `ranking` — fine-tune relevance scoring. See `https://pagefind.app/docs/ranking/`.
- `mergeIndex` — search across multiple Pagefind indexes. See `https://pagefind.app/docs/multisite/`.

## Alternative search: Algolia DocSearch

For users with access to Algolia's [DocSearch program](https://docsearch.algolia.com/), Starlight has an official plugin.

### Wiring

```bash
npm install @astrojs/starlight-docsearch
```

```js
// astro.config.mjs
import { defineConfig } from 'astro/config';
import starlight from '@astrojs/starlight';
import starlightDocSearch from '@astrojs/starlight-docsearch';

export default defineConfig({
  integrations: [
    starlight({
      title: 'Site with DocSearch',
      plugins: [
        starlightDocSearch({
          appId: 'YOUR_APP_ID',
          apiKey: 'YOUR_SEARCH_API_KEY',
          indexName: 'YOUR_INDEX_NAME',
        }),
      ],
    }),
  ],
});
```

The search bar now opens an Algolia modal instead of Pagefind's UI.

### Inline options

Pass these directly to `starlightDocSearch()`:

- `maxResultsPerGroup` — default `5`.
- `disableUserPersonalization` — default `false`. When `true`, recent searches and favorites are not saved to local storage.
- `insights` — default `false`. Enable the Algolia Insights plugin to send search events to your DocSearch index.
- `searchParameters` — pass-through object for [Algolia Search Parameters](https://www.algolia.com/doc/api-reference/search-api-parameters/).

### Function-style DocSearch options

For options like `transformItems()` or `resultsFooterComponent()` (functions, not serializable), use a separate config file:

```ts
// src/config/docsearch.ts
import type { DocSearchClientOptions } from '@astrojs/starlight-docsearch';

export default {
  appId: 'YOUR_APP_ID',
  apiKey: 'YOUR_SEARCH_API_KEY',
  indexName: 'YOUR_INDEX_NAME',
  getMissingResultsUrl({ query }) {
    return `https://github.com/algolia/docsearch/issues/new?title=${query}`;
  },
} satisfies DocSearchClientOptions;
```

```js
// astro.config.mjs
starlight({
  plugins: [
    starlightDocSearch({
      clientOptionsModule: './src/config/docsearch.ts',
    }),
  ],
})
```

See the [DocSearch JavaScript client API reference](https://docsearch.algolia.com/docs/api/) for all options.

### Translating the DocSearch UI

DocSearch ships only English strings. To add other languages, extend the `i18n` collection schema:

```ts
// src/content.config.ts
import { defineCollection } from 'astro:content';
import { docsLoader, i18nLoader } from '@astrojs/starlight/loaders';
import { docsSchema, i18nSchema } from '@astrojs/starlight/schema';
import { docSearchI18nSchema } from '@astrojs/starlight-docsearch/schema';

export const collections = {
  docs: defineCollection({ loader: docsLoader(), schema: docsSchema() }),
  i18n: defineCollection({
    loader: i18nLoader(),
    schema: i18nSchema({ extend: docSearchI18nSchema() }),
  }),
};
```

Then add translations to your JSON files in `src/content/i18n/`. The English defaults Starlight DocSearch ships with (use these as keys to override per-locale):

```
docsearch.searchBox.resetButtonTitle
docsearch.searchBox.resetButtonAriaLabel
docsearch.searchBox.cancelButtonText
docsearch.searchBox.cancelButtonAriaLabel
docsearch.searchBox.searchInputLabel
docsearch.startScreen.recentSearchesTitle
docsearch.startScreen.noRecentSearchesText
docsearch.startScreen.saveRecentSearchButtonTitle
docsearch.startScreen.removeRecentSearchButtonTitle
docsearch.startScreen.favoriteSearchesTitle
docsearch.startScreen.removeFavoriteSearchButtonTitle
docsearch.errorScreen.titleText
docsearch.errorScreen.helpText
docsearch.footer.selectText
docsearch.footer.selectKeyAriaLabel
docsearch.footer.navigateText
docsearch.footer.navigateUpKeyAriaLabel
docsearch.footer.navigateDownKeyAriaLabel
docsearch.footer.closeText
docsearch.footer.closeKeyAriaLabel
docsearch.footer.searchByText
docsearch.noResultsScreen.noResultsText
docsearch.noResultsScreen.suggestedQueryText
docsearch.noResultsScreen.reportMissingResultsText
docsearch.noResultsScreen.reportMissingResultsLinkText
```

## Community search providers

For users who want an open-source, self-hosted alternative to Algolia DocSearch, the community Starlight DocSearch Typesense plugin (`https://starlight-docsearch.typesense.org/`) integrates the DocSearch interface with a Typesense backend. The full list of community plugins is at `https://starlight.astro.build/resources/plugins/#community-plugins`.
